**Project context for design:**
This is a credit risk ML pipeline dashboard that shows performative drift detection in action. The audience is technical hiring managers at fintechs, MLOps platforms, and AI/ML teams. The dashboard must look like something a senior engineer built for a real production system — not a tutorial.

**Page structure:**
```
credit-risk-ml-pipeline
├── [sidebar]  Navigation: Overview | Drift Analysis | Model Registry | Alerts
└── [main]
    ├── metrics_header  — 4 KPI cards: Current Drift Score | Active Model | Alerts (30d) | Sim Day
    ├── drift_chart     — Line chart: drift score vs days, with red threshold line at 1.0, alert markers
    ├── [two columns]
    │   ├── model_history — Table: version, AUC, promoted_at, status (champion/challenger)
    │   └── alert_log     — Table: fired_at, drift_score, retrain triggered?, new model promoted?
    └── [footer]  "Phase 1 — placeholder data | Phase 6 will connect live PostgreSQL"
```

- Use `st.set_page_config(layout="wide", page_title="credit-risk", page_icon="📊")`
- Custom CSS via `st.markdown` for a dark-accent color scheme (dark navy sidebar, white main, red alert accents)
- Drift chart must use Plotly (not Altair/matplotlib) — add `plotly==5.24.1` to Dockerfile.dashboard install
- Metric cards use `st.metric()` with delta indicators
- Tables use `st.dataframe()` with column config for type-aware rendering
- Each component is a standalone function `render(data: list[dict]) -> None` in its own file
- No hardcoded database URLs — all data passed in as function arguments
- Add Google-style docstrings to all functions
