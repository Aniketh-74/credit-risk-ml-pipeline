<objective>
Build the custom MLflow Dockerfile and the full docker-compose.yml stack so all five services start with a single `docker compose up`.

Purpose: The official MLflow Docker image silently falls back to SQLite when psycopg2 is missing (confirmed GitHub issue #9513). This plan fixes that by building a custom image. The docker-compose.yml establishes the startup dependency graph that prevents race conditions between services — a known failure mode when `depends_on` is used without `condition: service_healthy`.

Output: docker/Dockerfile.mlflow, docker/Dockerfile.api, docker/Dockerfile.dashboard, docker-compose.yml, src/api/main.py (health stub).
</objective>

Service definitions:

**postgres** (image: postgres:16):
- environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB (all from env vars), POSTGRES_MULTIPLE_DATABASES=airflow_db,mlflow_db,app_db
- volumes: `./scripts/init-multiple-dbs.sh:/docker-entrypoint-initdb.d/init-multiple-dbs.sh:ro` and `postgres_data:/var/lib/postgresql/data`
- healthcheck: test `["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]`, interval: 10s, timeout: 10s, retries: 5, start_period: 30s
- ports: 5432:5432 (for local Alembic migrations run outside container)

**mlflow** (build: docker/Dockerfile.mlflow):
- depends_on: postgres: condition: service_healthy
- command: >
    mlflow server
    --backend-store-uri postgresql+psycopg2://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/mlflow_db
    --default-artifact-root /mlflow/artifacts
    --host 0.0.0.0
    --port 5000
- environment: MLFLOW_TRACKING_URI=http://mlflow:5000
- volumes: mlflow_artifacts:/mlflow/artifacts
- ports: 5000:5000
- healthcheck: test `["CMD-SHELL", "curl -f http://localhost:5000/health || exit 1"]`, interval: 15s, timeout: 10s, retries: 5, start_period: 45s

**airflow-init** (image: apache/airflow:3.1.8):
- depends_on: postgres: condition: service_healthy
- environment: AIRFLOW__DATABASE__SQL_ALCHEMY_CONN (from env var), AIRFLOW__CORE__FERNET_KEY (from env var), AIRFLOW__CORE__EXECUTOR=LocalExecutor
- command: db migrate
- Note: This is a one-shot init container. Other Airflow services depend on it with `condition: service_completed_successfully`

**airflow-scheduler** (image: apache/airflow:3.1.8):
