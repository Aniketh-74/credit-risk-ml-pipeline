# ADR-001: CB-PDD chosen over standard drift detectors

**Status:** Accepted  
**Date:** 2026-03-01  
**Deciders:** Aniketh Mahadik

---

## Context

The credit-risk scoring model operates in a performative setting: the model's own predictions influence applicant behaviour. Denied applicants return with nudged feature values; approved applicants exit the pool. Standard distribution-shift detectors (PSI, KL divergence, KSWIN) observe the resulting score distribution change but cannot distinguish two root causes:

1. **Natural covariate shift** — the population of applicants changes independently of the model.
2. **Performative feedback loop** — the model's denials cause denied applicants to re-apply with altered features, degrading the training distribution from the inside.

These two causes require different responses: covariate shift calls for data refresh, performative drift calls for bias-corrected retraining. A detector that cannot distinguish them will either trigger unnecessary retraining (false alarms on natural shift) or miss the feedback loop entirely.

---

## Decision

Use **CB-PDD** (CheckerBoard Performative Drift Detector, arXiv 2412.10545) as the primary drift signal, with PSI retained as a secondary baseline for comparison.

---

## Rationale

### Why CB-PDD

CB-PDD uses a controlled checkerboard prediction assignment — alternating approve/deny decisions for similar applicants across trial periods — to create a natural A/B structure. The density change rate `a = correction_rate(last_w) − correction_rate(first_w)` within each trial period captures the specific signature of denied applicants returning with improved scores. The Mann-Whitney U test on Group A vs Group B distributions detects whether this signature is statistically significant across consecutive windows.

Key properties:
- **Causally targeted**: detects the feedback-loop structure, not just any distribution shift
- **No external data required**: operates on (score, prediction, outcome) triples already in the predictions table
- **Two-consecutive-window gate**: requires 2 consecutive p < α windows, reducing false positives from single-trial noise
- **Parameter-transparent**: τ, w, α all exposed via environment variables with documented rationale

### Why not alternatives

| Detector | Reason rejected |
|----------|----------------|
| PSI only | Treats covariate shift and performative drift identically — cannot distinguish cause |
| KSWIN | Tests score distribution directly — same limitation as PSI |
| ADWIN | Concept drift on labels — requires outcome labels to arrive promptly; our label delay is 14–30 days |
| River/Evidently | External libraries add dependency weight and their algorithms are not tailored to performative settings |

### τ calibration

Three values were validated in Phase 2 (02-03 CB-PDD smoke test, denial loop simulation, n\_per\_day=1000):

| τ | First detection day | Notes |
|---|---------------------|-------|
| 500 | Day 6 | Fast but higher false-positive exposure |
| **1000** | **Day 14** | **Selected — balances speed and statistical power** |
| 2000 | Day 28 | Most conservative; appropriate for low-volume models |

τ=1000 was selected because it aligns with the paper's recommended default, provides 2000 samples of confirmation before alarming, and detected the denial-loop pattern reliably across all 30 simulation days.

---

## Consequences

**Positive:**
- Drift alerts are causally meaningful — retraining is triggered by the feedback loop, not by population noise
- PSI is retained as a secondary metric for comparison (visible in the dashboard CB-PDD vs PSI chart)
- The null-control experiment (random data does not fire) is in the test suite as a permanent regression guard

**Negative:**
- CB-PDD requires the checkerboard split-path router (10% of predictions routed through the checker), adding routing complexity to the scoring API
- Detection depends on label availability — if outcome labels arrive with >30 days delay, the rolling window may have insufficient labeled data
- τ must be recalibrated if daily prediction volume changes significantly from 1000/day
